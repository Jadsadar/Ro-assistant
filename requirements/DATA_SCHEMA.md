# RO Assistant Data Schema

This is the calculator-facing contract. The legacy calculator is the reference
for equipment slots, card positions, enchant slots, grade controls, and
character inputs.

## Static Catalog

`CatalogSearchItem` is the lightweight browser index:

```ts
interface CatalogSearchItem {
  id: number
  name: string
  aegisName: string
  itemTypeId: number
  itemSubTypeId: number
  itemType: string
  slot: string | null
  equipSlots: EquipmentSlot[]
  category: string
  slots: number
  compositionPos: number | null
  canGrade: boolean
  isRefinable: boolean
  requiredLevel: number | null
  searchable: string
}
```

- `equipSlots` is derived from the legacy `itemSubTypeId` rules.
- `compositionPos` filters card dropdowns without loading every item detail.
- `canGrade` controls whether the grade selector is visible.
- Descriptions, scripts, and combat fields remain in lazy detail chunks.
- `public/data/skills.json` is generated by instantiating the 40 legacy
  calculator classes and reading their real `atkSkills` metadata. The client
  loads this small file only on the Calculator route.

## Equipment Slots

Stable calculator slot IDs:

```text
weapon, leftWeapon, ammo, shield
headUpper, headMiddle, headLower
armor, garment, boot
accLeft, accRight, pet
costumeUpper, costumeMiddle, costumeLower, costumeGarment
costumeEnchantUpper, costumeEnchantMiddle, costumeEnchantLower
costumeEnchantGarment, costumeEnchantGarment2, costumeEnchantGarment4
shadowWeapon, shadowArmor, shadowShield
shadowBoot, shadowEarring, shadowPendant
```

Relationships are defined in `src/lib/equipment/types.ts`.

- Weapon: up to 4 cards, enchant slots 0-3.
- Shield, upper/middle headgear, armor, garment, boots, accessories:
  up to 1 card where supported, enchant slots 1-3.
- Lower headgear: no card, enchant slots 1-3.
- Shadow equipment: no cards, enchant slots 2-3.
- Costume and pet slots have no item relations in the legacy calculator.

## Item Variant

```ts
interface ItemVariant {
  id: string
  itemId: number
  fingerprint: string
  refine: number
  grade?: "D" | "C" | "B" | "A"
  cards: Array<{ slot: number; itemId: number }>
  enchants: ItemOption[]
  randomOptions: ItemOption[]
  customLabel?: string
  note?: string
  createdAt: string
  updatedAt: string
}

interface ItemOption {
  slot?: number
  key: string
  itemId?: number
  value?: number
  unit?: "flat" | "percent" | "level" | "text"
  label: string
}
```

- Cards reference a card catalog `itemId`.
- Known enchants retain the legacy enchant catalog `itemId`.
- Parsed random options use `key`, `value`, and `unit`.
- `label` remains available for display and unparsed options.
- Fingerprints include card IDs, enchant IDs, option values, refine, and grade.

Random options are selected from the legacy cascade values instead of free
text. Examples:

```json
{ "slot": 1, "key": "range", "value": 5, "unit": "percent", "label": "Long Range +5%" }
{ "slot": 2, "key": "str", "value": 10, "unit": "flat", "label": "STR +10" }
```

Weapons expose three random-option slots. Shadow equipment exposes one.
Other equipment uses the item-specific count from the legacy
`ExtraOptionTable`.

## Price Knowledge

```ts
interface PriceQuote {
  id: string
  variantId: string
  server: "thor" | "chaos"
  priceZeny: number
  quantity: number
  sourceType: "manual" | "market" | "shop" | "trade" | "import"
  sourceLabel?: string
  observedAt: string
  note?: string
  createdAt: string
  updatedAt: string
}
```

Price is an observation attached to a variant. Unknown price is represented by
the absence of a quote, never by zero.

## Saved Build

```ts
interface SavedBuild {
  id: string
  name: string
  classId: number
  className?: string
  baseLevel: number
  jobLevel: number
  skillId: string
  skillLevel: number
  propertyAtk: AttackElement
  monsterId: number
  server: "thor" | "chaos"
  equipment: Partial<Record<EquipmentSlot, string | null>>
  stats: Record<CharacterStatKey, number>
  skillLevels: Record<string, number>
  buffLevels: Record<string, number>
  consumableIds: number[]
  targetDamage?: number
  budgetZeny?: number
  createdAt: string
  updatedAt: string
}
```

Equipment values are `ItemVariant.id` references, not base catalog item IDs.

## Versions

- Catalog manifest schema: `2`
- Knowledge JSON export schema: `3`
- IndexedDB remains version `1`; its object-store layout did not change.

Future breaking field changes must increment the relevant schema version.
